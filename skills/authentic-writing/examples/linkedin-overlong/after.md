Stop satisfying the model. Start engineering the system.

We spent a year adopting AI tooling. The magic of one-off prompting didn't scale — outputs drifted, conventions eroded, review got painful.

Three changes fixed it:

→ A rule file the agent reads first (its day-one onboarding doc)
→ Verification gates, not vibes — nothing ships that can't pass tests and evals
→ One reviewer of record per change, so accountability never diffuses

The reviewer-of-record change alone did more for quality than anything else we tried.
