# Why a Rule File Matters

> TL;DR: A rule file is your agent's onboarding doc — write it once, save the drift.

The first time you hand a task to a coding agent, it behaves like a new engineer on day one — except it can't ask questions. A rule file is how you answer them in advance.

---

## Ad-hoc prompts vs. a rule file

Without a rule file, every prompt re-establishes context from scratch:

- **Ad-hoc prompts** — high cost (you re-explain conventions each time) and high drift (every session diverges).
- **Rule file** — low cost (context is loaded once) and low drift (everyone shares the same baseline).
